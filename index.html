Postmortem Claude Code Breaking – gestern

Anthropic gesteht: Drei Engineering-Fehler haben Claude Code einen Monat lang schlechter gemacht

Nach wochenlangem Nutzer-Backlash veröffentlichte Anthropic gestern (24. April) einen detaillierten Postmortem-Bericht. Drei separate Änderungen hatten Claude Code systematisch verschlechtert: (1) 4. März – Default-Reasoning-Effort von „high" auf „medium" reduziert, um Latenz zu senken. (2) 26. März – Bug, der die komplette Reasoning-History des Modells mid-Session löschte. (3) 16. April – System-Prompt, der Antworten auf 25 Wörter zwischen Tool-Calls cappte. Besonders brisant: Anthropic hatte zunächst kommuniziert, alles sei normal – Nutzer seien selbst schuld. Das Geständnis kommt, nachdem die Kritik öffentlich eskalierte. Alle drei Fehler wurden am 20. April behoben (v2.1.116). Zur Wiedergutmachung werden Nutzungslimits für alle Abonnenten zurückgesetzt.

Konsequenz: Anthropic verspricht strengere Qualitätssicherung und kündigt an, Änderungen an Modellverhalten künftig transparent zu kommunizieren.

OpenAI veröffentlicht GPT-5.5 – „ein neues Intelligenz-Niveau für echte Arbeit"

Nur sechs Wochen nach GPT-5.4 steht GPT-5.5 seit dem 23. April live für Plus-, Pro-, Business- und Enterprise-Nutzer zur Verfügung – sowohl in ChatGPT als auch in Codex. Das Modell (Codename „Spud") übertrifft seinen Vorgänger in Code-Debugging, Datenanalyse, autonomen Multi-Step-Workflows und Deep Research. GPT-5.5 übernimmt mehrstufige Aufgaben selbstständig – planen, Tools nutzen, Ergebnisse prüfen – ohne Step-by-Step-Prompting. Trotz gestiegenem Preis ist es laut OpenAI deutlich token-effizienter als GPT-5.4. In Codex liefert es bessere Resultate mit weniger Tokens. Zwei Varianten: Thinking (schneller für härtere Probleme) und Pro (höhere Aufgabenqualität). GPT-5.5 und GPT-5.5 Pro sind auch direkt über die API verfügbar.
